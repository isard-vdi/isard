Edit users from CSV


The CSV must be separated by commas or semicolons
You can upload up to 200 users per CSV
You can download a CSV file with the current users data using "CSV for update"
If the group of the user is changed:
  • Moving the users to another group will stop all of their desktops and empty their recycle bin
  • The users' resources could be affected by the new group's permissions
  • Surpassed quotas and group limits will not be checked
 CSV file should have these columns:
  • active / name / provider / category / uid / group / secondary_groups / password
  • Where category and group are their names
  • UID, category and provider's contents are mandatory
  • Secondary groups must be separated by "/"
  • Leave the parameters you don't want to update blank
  • 'Active' column must have as value either "true" or "false"

Important information when performing a migration:

  • Any running desktops will be stopped.
  • The user's recycle bin will be emptied.
  • The user's bookings will be deleted.
  • Non-persistent desktops will be DELETED
  • The list of co-owners in deployments will be deleted.
  • Desktops belonging to other users in migrated deployments will remain intact.
  • Resources (such as networks, RAM, CPU, GPUS) not allowed for the new user will be removed or restricted.
